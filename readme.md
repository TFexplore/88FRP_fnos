# 88FRP fnOS 项目说明

本仓库用于整理 `88FRP` 在飞牛 `fnOS` 场景下的应用实现、Node.js 版本重构以及相关开发文档。
本项目主要内容为:创建维护管理多个frpc配置，每个配置对应一个frpc实例，实例之间可以独立运行，互不干扰。另外可以设置88frp专用订阅密匙实现自动拉取同步配置重启frpc实例。


## 目录说明

### `88frp/`

这是当前可用于飞牛 `fnOS` 的完整版本，包含应用清单、安装脚本、Web UI、服务端逻辑以及打包所需文件，可直接作为飞牛应用包工程使用。

主要内容包括：

- `app/`：应用运行目录，包含 Web 页面、后端服务及运行时二进制
- `cmd/`：安装、卸载、升级、配置等生命周期脚本
- `config/`：权限与资源配置
- `wizard/`：安装或配置向导相关文件
- `manifest`：飞牛应用中心识别所需的应用描述文件

如果你的目标是：

- 打包飞牛应用
- 适配 `fnOS` 应用中心
- 维护当前已可用的飞牛版本

请优先查看和修改 `88frp/`。

### `88frp-node/`

这是基于 `Node.js` 开发的独立版本，用于将 `88FRP` 的核心能力逐步抽离为更易维护、可复用的实现。

当前目录主要包含：

- `src/core/`：实例管理、运行时管理、存储、同步等核心逻辑
- `src/cli/`：命令行入口
- `src/agent/`：后台常驻服务
- `src/web/`：Web 管理端服务与静态资源
- `scripts/`：构建与辅助脚本
- `tests/`：Node 版本相关测试

如果你的目标是：

- 开发基于 `Node.js` 的新版本
- 复用核心逻辑到 CLI、Web、Agent 等不同形态
- 构建单文件 Web 管理程序

请优先查看和修改 `88frp-node/`。

## 两个版本的关系

可以简单理解为：

- `88frp/`：当前完整的飞牛应用版本，偏向实际交付与封包
- `88frp-node/`：基于 `Node.js` 的重构版本，偏向核心逻辑拆分与后续演进

目前 `88frp/` 更适合作为 `fnOS` 应用包使用，`88frp-node/` 更适合作为后续功能迭代和架构演进的开发基础。

## 常用构建命令

### 1. 构建 Node 版本的单文件 Web 程序

在 `88frp-node/` 目录执行：

```bash
npm run build:web:single:linux-all
```

作用：

- 基于 `Node.js` 版本构建 Linux 平台单文件产物
- 同时生成 `linux-amd64` 与 `linux-arm64` 目标
- 适合后续集成到飞牛应用包中

### 2. 重新打包飞牛应用

在 `88frp/` 目录执行：

```powershell
Remove-Item -Path 88frp.fpk -Force
..\fnpack.exe build
```

作用：

- 删除旧的 `88frp.fpk` 安装包
- 使用 `fnpack` 重新生成飞牛应用安装包

如果你的 `fnpack` 不在上级目录，请按实际路径调整命令。

## 推荐工作流

如果你要维护飞牛可安装版本，推荐顺序如下：

1. 在 `88frp-node/` 中开发或调整 Node.js 逻辑
2. 生成需要集成的构建产物
3. 将产物或相关改动同步到 `88frp/`
4. 在 `88frp/` 中使用 `fnpack` 重新打包
5. 在飞牛 `fnOS` 环境中安装验证
