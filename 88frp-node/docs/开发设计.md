# 88FRP Node 版本开发设计

## 1. 项目定位

本项目用于实现 `88FRP` 的独立 `Node.js` 版本，目标是提供两种运行形态：

- 命令行版本：无网页前端，通过交互式命令菜单完成实例管理
- Docker 版本：提供网页端和后端服务，统一管理多个 `frpc` 实例

约束如下：

- 不修改 `D:\新建文件夹\88FRP_fnos\88frp` 原生 App 目录
- 新版本在同级目录单独创建项目并独立开发
- 命令行版本需要常驻后台程序，支持自动同步和启动时恢复实例运行状态

## 2. 总体架构

采用单项目多入口结构：

```text
88frp-node/
  docs/
    开发设计.md
  src/
    agent/
      server.js
    cli/
      index.js
    core/
      store.js
      process-manager.js
      instance-service.js
      runtime-service.js
      sync-service.js
    shared/
      constants.js
    web/
      server.js
  package.json
```

各层职责如下：

- `core`：公共业务核心，供 CLI、agent、web 共用
- `agent`：CLI 常驻后台服务，负责自动恢复、自动同步、统一进程管理
- `cli`：命令行菜单和子命令入口，通过本地 IPC 与 agent 通信
- `web`：Docker 版本的后端入口，对外提供 HTTP API 和静态页面托管能力

## 3. 核心原则

### 3.1 一套业务核心，多种入口

所有实例管理、配置存储、运行时控制、远程同步逻辑只在 `core` 内实现一次。

### 3.2 CLI 与 Docker 共享数据结构

统一数据目录结构：

```text
data/
  instances.json
  settings.json
  app.log
  instances/
    <instance-id>/
      frpc.toml
      runtime.json
      runtime.log
```

### 3.3 CLI 采用 agent 模式

命令行版本不是纯单次执行工具，而是：

- `88frpctl menu`：唤起交互菜单
- `88frpctl agent start`：启动后台 agent
- `88frpctl agent stop`：停止后台 agent
- `88frpctl agent status`：查看 agent 状态

原因：

- 自动同步需要定时任务
- 启动恢复需要在独立常驻进程内完成
- 多个命令入口都应复用同一份运行时状态

## 4. 组件设计

## 4.1 store

职责：

- 初始化数据目录
- 读写 `instances.json`
- 读写 `settings.json`
- 读写实例配置和运行时文件
- 读写应用日志和实例日志

## 4.2 process-manager

职责：

- 启动 `frpc` 进程
- 停止、重启实例
- 记录 PID 和运行状态
- 检查幽灵进程
- 在不同平台下处理进程退出

## 4.3 instance-service

职责：

- 创建实例
- 更新实例
- 删除实例
- 查询实例列表和单实例详情

## 4.4 runtime-service

职责：

- 启动实例
- 停止实例
- 重启实例
- 启动时恢复运行状态

## 4.5 sync-service

职责：

- 拉取远程配置
- 校验配置内容
- 比较配置差异
- 自动同步调度
- 配置变更后保存并决定是否重启实例

## 4.6 agent

职责：

- 常驻运行
- 启动时初始化数据目录
- 启动时恢复实例运行状态
- 周期执行自动同步
- 通过本地 IPC 接收 CLI 指令

第一阶段先使用本地 TCP `127.0.0.1` 作为 IPC，后续可演进为：

- Windows：Named Pipe
- Linux/macOS：Unix Domain Socket

## 4.7 web

职责：

- 提供 Docker 版本 HTTP API
- 对接网页端
- 复用 `core` 管理实例
- 对外暴露健康检查接口

## 5. CLI 交互模式

命令行版本包括两类交互：

### 5.1 菜单模式

用于人工操作：

- 查看实例列表
- 查看实例状态
- 创建实例
- 修改实例
- 删除实例
- 获取配置
- 设置配置
- 启动实例
- 停止实例
- 重启实例
- 手动同步
- 查看日志

### 5.2 子命令模式

用于脚本调用：

```bash
88frpctl agent start
88frpctl agent stop
88frpctl agent status
88frpctl list
88frpctl status <id>
88frpctl start <id>
88frpctl stop <id>
88frpctl restart <id>
88frpctl sync <id>
88frpctl logs <id>
```

## 6. agent 生命周期

启动流程：

1. 初始化数据目录
2. 校验 `frpc` 二进制路径
3. 清理残留运行时状态
4. 恢复需要运行的实例
5. 启动自动同步调度器
6. 监听本地 IPC 请求

停止流程：

1. 停止自动同步调度器
2. 关闭 IPC 监听
3. 视策略决定是否停止全部实例
4. 写入退出日志

## 7. Docker 版本设计

Docker 版本采用单容器多实例管理模式：

- 容器内运行 Node Web 服务
- 同一容器管理多个 `frpc` 实例
- 数据目录挂载到卷
- 前端页面通过 HTTP API 访问后端

推荐环境变量：

| 变量名 | 说明 |
| --- | --- |
| `PORT` | Web 监听端口 |
| `HOST` | Web 监听地址 |
| `DATA_DIR` | 数据目录 |
| `FRPC_BINARY_PATH` | `frpc` 可执行文件路径 |
| `AUTO_SYNC_INTERVAL_MS` | 自动同步间隔 |
| `INSTANCE_AUTO_START_ON_BOOT` | 是否启动时恢复实例 |
| `AGENT_PORT` | 本地 agent 监听端口 |

当前已确定：

- 项目单独放在 `88frp-node`
